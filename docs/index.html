Claude Code Starter Kit

Claude Code の主要機能を網羅したスターターキット

Claude Code の導入を検討しているチームや個人に向けたスターターキットです。Skills、カスタムコマンド、サブエージェント、エージェントチームなど、Claude Code の主要機能をすぐに導入・使用できるテンプレートとサンプルを提供します。

前提条件

  • Claude Code CLI がインストール済みであること
  • Node.js 18 以上
  • Git
# Claude Code のインストール
npm install -g @anthropic-ai/claude-code

# インストール確認
claude --version

はじめ方

3ステップですぐに使い始められます。

  1. クローン - リポジトリをクローンします 
    git clone https://github.com/sas-dx/claude-code-starter-kit.git
cd claude-code-starter-kit
  2. 起動 - Claude Code を起動すると CLAUDE.md が自動的に読み込まれます 
    claude
  3. カスタマイズ - プロジェクトに合わせて CLAUDE.md を編集します 
    # 自動生成してからカスタマイズ
claude /init

/scaffold-feature スキルで各機能の雛形を一括生成できます。例: /scaffold-feature command deploy

主要機能

📋

CLAUDE.md

セッション開始時に自動読み込みされるプロジェクト指示書。チーム全員が同じルールで作業でき、AI の暴走も防止します。
📏

Rules

CLAUDE.md の詳細版として機能するルールファイル。パス別ルールで特定ファイル操作時のみ適用させることも可能です。
⌨️

Commands

よく行う作業を /コマンド名 の一言で実行。コードレビュー、Issue ベース実装、コンフリクト解決などを提供。
🎯

Skills

コマンドの上位互換。手動呼び出しに加え、Claude が状況に応じて自動的に使用。テンプレートやスクリプトの同梱にも対応。
🤖

Agents

特定タスクに特化したサブエージェント。隔離コンテキストで独立動作し、Claude が必要に応じて自動的に委譲します。
👥

Agent Teams

複数の Claude Code インスタンスが並列で協調作業を行う実験的機能。共有タスクリストで自律的に協調します。
🪝

Hooks

ツール実行前後に自動実行されるシェルスクリプト。CLAUDE.md が「指示」ならば、Hooks は「仕組みで強制する」ガードレールです。
🔌

MCP

AI に GitHub、DB、Slack 等の外部サービスへのアクセスを提供する仕組み。Serena や Context7 を導入済み。
🧩

Plugins

Skills・Agents・Hooks・MCP サーバー等を1つのパッケージにまとめて配布・共有できる仕組みです。
⚙️

Settings

Claude Code の動作・権限・フック・環境変数を制御する設定ファイル。優先度付きの階層構造で管理します。
⌨️

Keybindings

キーボードショートカットのカスタマイズ。/keybindings で設定ファイルを開いて編集できます。
🏗️

AI駆動開発アーキテクチャ

基幹システムリプレイスを題材に、Claude Code の全機能を組み合わせたチーム開発のグランドデザインを解説します。

ディレクトリ構成

.claude/ ディレクトリに Claude Code の全設定とカスタマイズが集約されています。

claude-code-starter-kit/
├── README.md
├── .mcp.json                        # MCP サーバー設定（チーム共有）
└── .claude/
    ├── CLAUDE.md                    # プロジェクト指示書（メイン）
    ├── README.md                    # CLAUDE.md の書き方ガイド
    ├── settings.json                # プロジェクト設定（チーム共有）
    ├── commands/                    # カスタムコマンド
    ├── skills/                      # スキル
    ├── agents/                      # サブエージェント
    ├── rules/                       # プロジェクトルール
    ├── hooks/                       # フックスクリプト
    ├── mcp/                         # MCP ガイド
    ├── agent-teams/                 # Agent Teams ガイド
    ├── settings/                    # Settings ガイド
    ├── keybindings/                 # Keybindings ガイド
    └── plugins/                     # Plugins ガイド

各ディレクトリには README.md とサンプルファイルが含まれています。

カスタマイズガイド

1. プロジェクトに合わせた CLAUDE.md の作成

# 自動生成してからカスタマイズ
claude /init

2. 機能の追加

/scaffold-feature スキルで雛形を一括生成できます。

/scaffold-feature command deploy       # カスタムコマンドの雛形
/scaffold-feature agent test-runner    # サブエージェントの雛形
/scaffold-feature skill my-workflow    # スキルの雛形
/scaffold-feature rule api-design      # ルールの雛形
/scaffold-feature hook block-rm        # フックスクリプトの雛形

3. MCP サーバーの追加

claude mcp add --transport http --scope project <name> <url>

一番大切なこと

AI 駆動開発はまだ黎明期です。Claude Code をはじめとする AI ツールは日々進化しており、今後も新機能が登場し、より洗練されていくことが予想されます。

このスターターキットも完成形ではありません。大切なのは以下のサイクルを回し続けることです。

調べる → 試す → 良かったものを共有する → プロジェクトに組み込む
  1. 調べる - 時間があるときに公式ドキュメントやリリースノート、コミュニティの事例を確認する
  2. 試す - 新しい機能や方法論を実際に使ってみる。うまくいかなくても経験になる
  3. 共有する - 良かったものはチームや他の案件に横展開する。スキル・ルール・エージェントとして形式知化する
  4. 組み込む - 自分が所属するプロジェクトやリポジトリを、より AI 駆動開発に適した環境に整備する

ツールに詳しい1人のエンジニアが全てを整備するのではなく、チーム全員が少しずつ改善を積み重ねることで、プロジェクト全体の開発体験が向上していきます。
