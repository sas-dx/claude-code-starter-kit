はじめに（導入ガイド）

このページでは、Claude Code をゼロからセットアップして使い始めるまでの全手順を、新入社員でも迷わないレベルの詳しさで解説します。各ステップで「なぜこの手順が必要か」を補足し、エラーが出た場合の対処法もあわせて掲載しています。

Claude Code とは

Claude Code の概要:
  • Claude Code は Anthropic が提供する AI コーディングアシスタントの CLI ツールです
  • ターミナルから起動し、対話形式でコードの生成・編集・レビュー・テスト実行などを行います
  • プロジェクトのファイルを直接読み書きでき、Git 操作やコマンド実行も可能です
  • VS Code や JetBrains IDE の拡張機能としても利用できます

従来の開発との違い

Claude Code を使った開発は、従来の開発とは根本的にワークフローが異なります。

従来の開発

開発者がエディタを開き、コードを1行ずつ手で書く。ドキュメントやStack Overflowを検索しながら進める。テストも手動で書き、レビューも人間が目視で行う。

Claude Code を使った開発

「この関数にバリデーションを追加して」と日本語で指示するだけで、AI がコードを生成・編集する。人間は設計判断とレビューに集中できる。テスト作成やリファクタリングも AI に任せられる。

Claude Code がどのような位置づけのツールなのか、全体像を図で確認しましょう。開発者がターミナルや IDE から Claude Code を起動すると、AI がコード生成、レビュー、Git 操作、外部ツール連携といった幅広い作業を代行してくれます。

flowchart LR USER["開発者"] --> CLAUDE["Claude Code
(CLI / IDE)"] CLAUDE --> CODE["コード生成・編集"] CLAUDE --> REVIEW["レビュー・テスト"] CLAUDE --> GIT["Git 操作"] CLAUDE --> TOOLS["外部ツール連携
(MCP)"]

このように、Claude Code は単なるコード補完ツールではなく、開発ワークフロー全体をサポートするパートナーです。ここからは、このツールを使い始めるための手順を一つずつ進めていきましょう。

Claude Code でできること

具体的にどのようなことができるのか、代表的なユースケースを紹介します。

ユースケース 具体例 従来の方法
コード生成 「ユーザー登録の API エンドポイントを作成して」と指示するだけで、ルーティング・バリデーション・DB 操作を含むコードを生成 ドキュメントを読みながら手動で実装
バグ修正 エラーメッセージを貼り付けて「この原因を調べて修正して」と依頼 スタックトレースを読み解き、デバッガで追跡
コードレビュー 「PR #42 のコードをレビューして」で、セキュリティ・パフォーマンス・規約準拠を自動チェック レビュアーが1行ずつ目視確認
テスト作成 「この関数のユニットテストを書いて」で、正常系・異常系のテストケースを自動生成 テストフレームワークのドキュメントを見ながら手動作成
リファクタリング 「この関数を小さな関数に分割して」で、責務分離されたコードに変換 設計を考えながら慎重に手動変更
ドキュメント生成 「この API の仕様書を Markdown で書いて」で、自動的にドキュメントを生成 コードを読みながら手動で文書化

前提条件の確認

このステップが必要な理由: Claude Code は Node.js 上で動作するツールです。また、プロジェクト管理に Git を使用します。まず、これらがインストールされているか確認しましょう。前提条件が揃っていないと、後続のインストール手順でエラーが発生します。

2.1 Node.js の確認

Node.js は JavaScript をサーバーサイドで動かすための実行環境です。Claude Code は Node.js のパッケージマネージャー（npm）を通じてインストールするため、Node.js が必須です。

以下のコマンドをターミナルで実行して、Node.js がインストールされているか確認します。

# このコマンドは Node.js のバージョンを表示します
node --version
# v18.0.0 以上が表示されれば OK
command not found と表示された場合: Node.js がインストールされていません。以下の手順でインストールしてください。
  • 公式サイトからダウンロード（全 OS 共通・推奨）: https://nodejs.org/ から LTS 版（長期サポート版）をダウンロードしてインストールします。LTS 版は安定性が高く、企業利用に適しています。
  • macOS（Homebrew を利用）: 
    # Homebrew がインストール済みであれば、以下のコマンドで Node.js をインストールできます
brew install node
  • Windows: 公式サイトからインストーラー（.msi）をダウンロードし、画面の指示に従ってインストールします。インストール完了後、ターミナル（PowerShell やコマンドプロンプト）を再起動してください。
  • Ubuntu / WSL: 
    # NodeSource リポジトリから Node.js 20.x をインストールします
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

インストール後、もう一度 node --version を実行して、バージョンが表示されることを確認してください。

2.2 npm の確認

npm（Node Package Manager）は Node.js のパッケージ管理ツールです。Claude Code を含む世界中の JavaScript パッケージをインストール・管理できます。

# このコマンドは npm のバージョンを表示します
npm --version
# 9.0.0 以上が表示されれば OK
ヒント: npm は Node.js に同梱されています。Node.js をインストールすれば自動的にインストールされます。npm だけが見つからないという状況は通常発生しませんが、もし見つからない場合は Node.js のインストールが正常に完了していない可能性があります。Node.js を再インストールしてください。

2.3 Git の確認

Git はバージョン管理システムです。Claude Code はプロジェクトの変更履歴を Git で管理し、ブランチ作成やコミットなどの操作も行います。

# このコマンドは Git のバージョンを表示します
git --version
# git version 2.x.x が表示されれば OK
command not found と表示された場合: Git がインストールされていません。以下の手順でインストールしてください。
  • macOS: 
    # Xcode コマンドラインツールをインストールすると Git も含まれます
xcode-select --install

# または Homebrew を利用
brew install git
  • Windows: https://git-scm.com/ から Git for Windows をダウンロードしてインストールします。インストール時の設定は基本的にデフォルトのままで問題ありません。
  • Ubuntu / WSL: 
    # apt パッケージマネージャーで Git をインストールします
sudo apt-get update
sudo apt-get install -y git

2.4 まとめて確認

以下のコマンドを実行して、3つのツールが全てインストールされていることを一度に確認できます。

# 3つのバージョンを一度に確認するコマンドです
# && は「前のコマンドが成功したら次を実行する」という意味です
node --version && npm --version && git --version

3行のバージョン番号が全て表示されれば、前提条件はクリアです。次のステップに進みましょう。

Claude Code のインストール

なぜ npm でインストールするのか: Claude Code は npm パッケージとして配布されています。-g フラグをつけることで、どのディレクトリからでも claude コマンドが使えるようになります。npm のグローバルインストールは「システム全体で使えるコマンド」を追加する標準的な方法です。

以下のコマンドを実行して、Claude Code をインストールします。

# Claude Code をグローバルインストールします
npm install -g @anthropic-ai/claude-code

このコマンドの各部分の意味は以下の通りです。

部分 意味
npm install npm パッケージをインストールするコマンドです
-g グローバルインストールを意味します。このフラグがないとカレントディレクトリにしかインストールされません
@anthropic-ai/claude-code Claude Code のパッケージ名です。@anthropic-ai は Anthropic 社の npm スコープ（名前空間）です

インストールの確認

インストールが完了したら、以下のコマンドで正しくインストールされたか確認します。

# Claude Code のバージョンを表示します
claude --version

バージョン番号が表示されれば、インストール成功です。

EACCES: permission denied エラーが出た場合: npm のグローバルインストールに管理者権限が必要なことがあります。以下のいずれかの方法で解決できます。
  • 方法 1: sudo を付けて実行する（手軽） 
    # 管理者権限で実行します（パスワードの入力を求められます）
sudo npm install -g @anthropic-ai/claude-code
  • 方法 2: npm のグローバルディレクトリを変更する（推奨） 
    # ホームディレクトリに npm グローバル用のディレクトリを作成します
mkdir -p ~/.npm-global

# npm のグローバルディレクトリを変更します
npm config set prefix '~/.npm-global'

# PATH に追加します（~/.bashrc または ~/.zshrc に追記）
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 再度インストールを実行します
npm install -g @anthropic-ai/claude-code
command not found: claude と表示された場合: npm のグローバルバイナリへのパスが通っていない可能性があります。以下の手順で確認・修正してください。
  1. npm のグローバルバイナリのパスを確認します: 
    # このコマンドは npm グローバルインストール先のパスを表示します
npm config get prefix
  2. 表示されたパスに /bin を追加して、シェルの設定ファイルに追記します: 
    # 例: npm config get prefix が /usr/local と表示された場合
# ~/.bashrc（Bash の場合）または ~/.zshrc（Zsh の場合）に以下を追記
export PATH="/usr/local/bin:$PATH"

# 設定を反映します
source ~/.bashrc   # Bash の場合
source ~/.zshrc    # Zsh の場合
  3. ターミナルを再起動してから claude --version を再度実行します。

4.1 Claude Code の起動

任意のディレクトリ（プロジェクトフォルダが望ましい）でターミナルを開き、以下のコマンドを実行します。

# Claude Code を起動します
# プロジェクトのルートディレクトリで実行するのがおすすめです
claude
初回起動時の動作: 初回起動時はブラウザが自動で開き、Anthropic アカウントへのログインを求められます。アカウントをまだ持っていない場合は、画面の指示に従って新規作成してください。2回目以降の起動では、認証情報が保持されているため自動的にログインされます。

4.2 認証方法

Claude Code には 3 つの認証方法があります。状況に応じて最適な方法を選択してください。

方法 1: ブラウザ認証（推奨）

最も簡単な方法です。初回起動時にブラウザが自動で開くので、画面の指示に従ってログインするだけです。

# ブラウザ認証（自動）
# ターミナルで claude を実行すると、ブラウザが自動的に開きます
claude
# → ブラウザが開くのでログインしてください
# → ログイン完了後、ターミナルに戻ると使用可能になっています

方法 2: API キーで認証

CI/CD 環境やサーバーなど、ブラウザが使えない環境では API キーを使います。Anthropic Console で API キーを発行してください。

# 環境変数に API キーを設定してから起動します
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxx"
claude

方法 3: 起動時に API キーを直接渡す

一時的に使用する場合や、テスト目的で使用する場合に便利です。

# 起動時にキーを渡します
claude --api-key "sk-ant-xxxxxxxxxxxxx"
API キーを毎回入力したくない場合: シェルの設定ファイルに環境変数を追記しておくと、ターミナル起動時に自動で読み込まれます。 
# ~/.bashrc（Bash の場合）または ~/.zshrc（Zsh の場合）に追記
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxx"

# 設定を反映します
source ~/.bashrc
API キーは秘密情報です。 絶対に以下の行為をしないでください。
  • Git リポジトリにコミットする（.env ファイルに書いて .gitignore で除外するのは OK）
  • Slack やチャットで共有する
  • ソースコードにハードコーディングする
  • 公開リポジトリの Issue やコメントに記載する

万が一漏洩した場合は、Anthropic Console から即座にキーを無効化し、新しいキーを発行してください。

4.3 初回起動の画面説明

認証が完了すると、ターミナルに Claude Code の対話画面が表示されます。主な要素は以下の通りです。

画面要素 説明
プロンプト（入力欄） > マークが表示されている箇所です。ここに質問や指示を入力します
モデル表示 現在使用している AI モデル名が表示されます
プロジェクト情報 カレントディレクトリに CLAUDE.md がある場合、自動的に読み込まれた旨が表示されます
困ったら /help: 対話画面で /help と入力すると、利用可能なコマンドや使い方のヘルプが表示されます。操作に迷ったらまず /help を試してください。

基本操作

Claude Code が起動したら、さっそく基本的な操作を試してみましょう。Claude Code は自然言語（日本語 OK）で指示を出すだけで、さまざまな作業を実行できます。

5.1 質問する

最もシンプルな使い方です。コードについて質問すると、Claude が回答してくれます。

> この関数は何をしていますか？
> このエラーメッセージの原因を教えてください
> React の useEffect フックの使い方を説明してください
ヒント: 日本語でも英語でも、どちらでも質問できます。プロジェクトの CLAUDE.md で「回答は日本語で」と指定しておけば、常に日本語で回答してくれます。

5.2 ファイルを読む・編集する

Claude Code はプロジェクト内のファイルを直接読み書きできます。ファイルパスを指定して指示するだけです。

# ファイルの内容を確認する
> src/index.ts を読んで内容を教えてください

# ファイルを編集する
> src/utils.ts の calculateTotal 関数にエラーハンドリングを追加してください

# 新しいファイルを作成する
> src/helpers/validation.ts にメールアドレスのバリデーション関数を作成してください
ファイル編集の安全性: Claude Code がファイルを編集する前に、変更内容の差分（diff）が表示されます。内容を確認してから承認することで、意図しない変更を防ぐことができます。

5.3 コマンドを実行する

テストの実行やビルドなど、ターミナルコマンドの実行も Claude Code に任せられます。

# テストを実行する
> npm test を実行してください

# ビルドする
> npm run build を実行してください

# lint を実行する
> eslint で src/ 配下をチェックしてください
コマンド実行の承認: Claude Code がシステムコマンドを実行する際には、事前に確認プロンプトが表示されます。内容を確認してから承認してください。安全でないコマンド（データ削除など）は自動では実行されません。Settings で自動承認するコマンドを設定することも可能です。

5.4 Git 操作

Claude Code は Git 操作もサポートしています。ブランチの作成、コミット、差分の確認などを自然言語で指示できます。

# 変更の確認
> git diff を見せてください

# コミット
> 現在の変更を「ログイン機能のバリデーションを追加」というメッセージでコミットして

# ブランチ作成
> feature/user-auth というブランチを作成して切り替えて

# 差分の要約
> main ブランチとの差分を要約してください
CLAUDE.md との連携: スターターキットの CLAUDE.md には「コミットメッセージは日本語で記述」「本番ブランチに直接プッシュしない」といったルールが定義されています。Claude Code はこれらのルールに従って Git 操作を行うため、チーム全体で統一されたワークフローを維持できます。

5.5 スラッシュコマンド

Claude Code にはスラッシュ（/）で始まる組み込みコマンドがあります。よく使う操作を素早く実行できます。

コマンド 説明 使用場面
/help ヘルプを表示します 操作に迷ったとき
/clear 会話履歴をクリアします 新しい話題に切り替えるとき
/init CLAUDE.md を自動生成します プロジェクトの初期設定
/model 使用モデルを切り替えます モデルを変更したいとき
/cost 現在のセッションのコストを表示します 利用料金を確認したいとき
/compact 会話を要約してコンテキストを圧縮します 長い会話でトークンを節約したいとき
/doctor インストール状態を診断します 動作に問題があるとき
ヒント: / を入力するだけで、利用可能なコマンドの一覧が表示されます。カスタムコマンドが設定されていれば、それも一覧に表示されます。

5.6 終了方法

Claude Code を終了するには、以下のいずれかの方法を使います。

# 方法 1: スラッシュコマンドで終了
/exit

# 方法 2: キーボードショートカットで終了
# Ctrl+C を押す（2回押す場合もあります）

# 方法 3: 自然言語で終了
> 終了してください

スターターキットの使い方

Claude Code Starter Kit は、チームですぐに Claude Code を使い始められるテンプレート集です。CLAUDE.md やルール、カスタムコマンドなどがあらかじめ整備されており、クローンして起動するだけで使えます。

flowchart TD S1["1. リポジトリをクローン"] S2["2. ディレクトリに移動"] S3["3. Claude Code を起動"] S4["4. CLAUDE.md が自動読み込み"] S5["5. 各機能がすぐに使える"] S1 --> S2 --> S3 --> S4 --> S5

クローンから起動まで

# 1. リポジトリをクローンします
# このコマンドは GitHub からスターターキットのソースコード一式をダウンロードします
git clone https://github.com/sas-dx/claude-code-starter-kit.git

# 2. クローンしたディレクトリに移動します
cd claude-code-starter-kit

# 3. Claude Code を起動します
# .claude/CLAUDE.md が自動的に読み込まれ、ルールが適用されます
claude
自動で適用される設定: .claude/CLAUDE.md が自動的に読み込まれ、プロジェクトのルール（コミットメッセージは日本語、DB 削除禁止など）が適用されます。何も設定しなくても、チーム全員が同じルールで作業できる状態になります。さらに .claude/rules/ 配下のルールファイルや .claude/settings.json の設定も自動で反映されます。

ディレクトリ構成

スターターキットの .claude/ ディレクトリには、以下のような構成でファイルが配置されています。各ディレクトリの役割を理解しておくと、カスタマイズがスムーズに進みます。

.claude/ ディレクトリ構成
.claude/
├── CLAUDE.md          # プロジェクト指示書（最も重要な設定ファイル）
├── settings.json      # 権限・環境の設定
├── rules/             # トピック別のルールファイル
│   ├── coding-standards.md   # コーディング規約
│   ├── git-workflow.md       # Git ワークフロー
│   ├── security.md           # セキュリティルール
│   ├── testing.md            # テストルール
│   └── review-process.md     # コードレビュールール
├── commands/          # カスタムスラッシュコマンド
│   ├── code-review.md        # /code-review コマンド
│   ├── dev-issue.md          # /dev-issue コマンド
│   └── ...
├── agents/            # サブエージェント定義
│   ├── code-reviewer.md      # コードレビュー専門エージェント
│   └── ...
└── skills/            # スキル定義
    └── ...            # 専門知識の形式知化
どこから手をつけるべきか: まずは CLAUDE.md を自分のプロジェクトに合わせて編集してください。次に rules/ 配下のルールをプロジェクトのコーディング規約に合わせます。カスタムコマンドやエージェントは必要に応じて追加・修正してください。

起動後に試せること

スターターキットには、すぐに試せるカスタムコマンドやサブエージェントが含まれています。

# カスタムコマンドを試す
/code-review #1          # PR やコードのレビューを実行します
/explain-code src/app.ts  # 指定したファイルのコードを解説します
/dev-issue #42            # GitHub Issue に基づいて実装を行います

# CLAUDE.md を自分のプロジェクト用にカスタマイズ
/init                     # 現在のプロジェクトに合わせて CLAUDE.md を自動生成します
自分のプロジェクトに適用するには: スターターキットの .claude/ ディレクトリを自分のプロジェクトにコピーし、CLAUDE.md の内容をプロジェクトに合わせて編集するだけで、同じ仕組みを導入できます。

既存プロジェクトへの導入手順

スターターキットを参考に、既存のプロジェクトに Claude Code の設定を導入する手順です。

# 1. 自分のプロジェクトのルートディレクトリに移動
cd /path/to/your-project

# 2. スターターキットの .claude ディレクトリをコピー
cp -r /path/to/claude-code-starter-kit/.claude .

# 3. CLAUDE.md をプロジェクトに合わせて編集
#    技術スタック、開発コマンド、ディレクトリ構造を自分のプロジェクトの情報に書き換えます
vi .claude/CLAUDE.md

# 4. .claude ディレクトリを Git に追加（チーム共有のため）
git add .claude/
git commit -m "Claude Code の設定ファイルを追加"

# 5. Claude Code を起動して動作確認
claude
settings.local.json は Git にコミットしないでください。 .claude/settings.local.json には個人のトークンや環境固有の設定が含まれることがあります。.gitignore に .claude/settings.local.json を追加しておくことを推奨します。

IDE 連携（VS Code / JetBrains）

Claude Code はターミナル（CLI）から使うだけでなく、普段使っている IDE（統合開発環境）の拡張機能としても利用できます。CLI と IDE 拡張は同じ設定ファイルを共有するため、どちらから使っても同じルールが適用されます。

7.1 VS Code

VS Code で Claude Code を使うには、拡張機能をインストールします。

方法 1: コマンドラインからインストール

# このコマンドは VS Code の拡張機能をターミナルからインストールします
code --install-extension anthropic.claude-code

方法 2: VS Code のマーケットプレイスからインストール

  1. VS Code を開きます
  2. 左サイドバーの拡張機能アイコン（四角が4つ並んだマーク）をクリックします
  3. 検索バーに「Claude Code」と入力します
  4. Anthropic が提供する拡張機能を見つけて「インストール」をクリックします
code コマンドが見つからない場合: VS Code を開き、コマンドパレット（Ctrl+Shift+P / Cmd+Shift+P）で「Shell Command: Install 'code' command in PATH」を実行してください。これにより、ターミナルから code コマンドが使えるようになります。

7.2 JetBrains（IntelliJ, WebStorm 等）

JetBrains 系の IDE（IntelliJ IDEA, WebStorm, PyCharm, PhpStorm 等）でも Claude Code を使用できます。

  1. IDE を開きます
  2. Settings（Ctrl+Alt+S / Cmd+,）を開きます
  3. Plugins > Marketplace に移動します
  4. 「Claude Code」で検索し、インストールします
  5. IDE を再起動します
CLI と IDE 拡張の設定共有: CLI と IDE 拡張は同じ .claude/CLAUDE.md、.claude/rules/、.claude/settings.json を参照します。CLI で設定した内容は IDE 拡張にもそのまま反映されるため、設定を二重管理する必要はありません。

よくあるトラブルと対処法

導入時によく遭遇するトラブルとその対処法をまとめました。問題が発生した場合は、まずこの一覧を確認してください。

症状 原因 対処法
command not found: claude npm のグローバルバイナリへのパスが通っていない npm config get prefix でパスを確認し、PATH 環境変数に （表示されたパス）/bin を追加してください
EACCES: permission denied npm のグローバルインストールに権限が不足している sudo を付けて再実行するか、npm のグローバルディレクトリをホームディレクトリ配下に変更してください（インストールセクション参照）
Error: API key not found 認証が完了していない、または API キーが設定されていない claude を再起動してブラウザ認証を行うか、ANTHROPIC_API_KEY 環境変数を設定してください
Error: Rate limit exceeded API の利用上限に達した しばらく待ってからリトライしてください。頻発する場合は Anthropic Console でプランを確認してください
ブラウザが開かない ヘッドレス環境（SSH接続先、Docker コンテナ等）で実行している claude --api-key "sk-ant-xxx" で API キーを直接指定して認証してください
日本語が文字化けする ターミナルの文字コードが UTF-8 になっていない ターミナルの設定で文字コードを UTF-8 に変更してください。Windows の場合は chcp 65001 を実行してからお試しください
CLAUDE.md が読み込まれない ファイルの配置場所が間違っている プロジェクトルート直下の CLAUDE.md、または .claude/CLAUDE.md に配置してください。Claude Code はこの 2 箇所を自動的に探索します
Node.js のバージョンが古い v18 未満の Node.js がインストールされている Node.js 公式サイトから LTS 版をダウンロードして上書きインストールしてください
それでも解決しない場合: claude /doctor コマンドを実行すると、インストール状態の診断が行われます。問題の特定に役立ちます。また、Claude Code の GitHub Issues で同様の問題が報告されていないか検索してみてください。

次のステップ

Claude Code の導入が完了しました。ここからは、チームの開発ワークフローに合わせて Claude Code をカスタマイズしていきましょう。以下のページで各機能の詳細を学べます。

📋 CLAUDE.md

プロジェクト指示書の書き方を学びます。Claude Code に「どのプロジェクトで、どんなルールで作業するか」を伝えるための最も重要な設定ファイルです。まずはここから始めましょう。

📏 Rules

コーディング規約やセキュリティルールを定義する方法を学びます。CLAUDE.md を補完し、トピック別・パス別にルールを管理できます。

⌨️ Commands

カスタムスラッシュコマンドの作り方を学びます。コードレビューや Issue 対応など、よく使う操作を /コマンド名 の一言で実行できるようになります。

🏗️ AI駆動開発アーキテクチャ

Claude Code の全機能を組み合わせた実践ガイドです。実際のプロジェクトでどう活用するか、基幹システムリプレイスを題材に解説しています。

おすすめの学習順序: まず CLAUDE.md でプロジェクト指示書の書き方を学び、次に Rules でルール管理を理解し、最後に Commands で作業の自動化に進むのがスムーズです。全体像を先に把握したい場合は AI駆動開発アーキテクチャ から読み始めるのも有効です。
