# セキュリティルール

## 禁止事項

### 絶対にやってはいけないこと

- **データベースの削除・初期化コマンドを実行しない**（`flush`、`drop`、`reset` 等）
- **機密情報をソースコードにハードコーディングしない**（API キー、パスワード、シークレット）
- **`.env` ファイルをコミットしない**
- **本番ブランチに直接プッシュしない**

## セキュリティチェックリスト

コード実装時に以下の観点を確認すること。

### 認証・認可

- [ ] 全ての API エンドポイントに認証チェックがあるか
- [ ] ユーザーの権限に基づいたアクセス制御が実装されているか
- [ ] セッション管理が安全か（トークンの有効期限、安全な保存先）
- [ ] パスワードは適切にハッシュ化されているか

### インジェクション対策

- [ ] SQL クエリに ORM を使用しているか（生 SQL の場合はパラメータバインディング）
- [ ] ユーザー入力をそのまま HTML に出力していないか（XSS 対策）
- [ ] CSRF トークンが適切に使用されているか
- [ ] シェルコマンドにユーザー入力を直接渡していないか

### データ保護

- [ ] API レスポンスに不要な機密データが含まれていないか
- [ ] ログに機密情報（パスワード、トークン等）が出力されていないか
- [ ] エラーメッセージからシステム内部の情報が漏洩していないか

### 入力バリデーション

- [ ] 全てのユーザー入力に対してバリデーションが実装されているか
- [ ] ファイルアップロードに拡張子・サイズの制限があるか
- [ ] パストラバーサル攻撃を防ぐチェックがあるか

### 依存関係

- [ ] 既知の脆弱性を持つパッケージを使用していないか
- [ ] 依存パッケージのバージョンが適切に管理されているか

## 機密情報の管理

### 環境変数を使用する

```
悪い例:
  const API_KEY = 'sk-abc123...';
  const DB_PASSWORD = 'mypassword';

良い例:
  const API_KEY = process.env.API_KEY;
  const DB_PASSWORD = process.env.DB_PASSWORD;
```

### .gitignore に含めるべきファイル

```
.env
.env.local
.env.production
*.pem
*.key
credentials.json
```
